## Migrating from Pterodactyl (Docker)

<Callout type='error'>
Once you switch to Hydrodactyl, switching back to Pterodactyl is not supported and may cause errors as Hydrodactyl modifies the Pterodactyl Database Schema. Provided below is how you can make a backup of your Panel Database if you need to switch back. (Replace ``panel`` with the database name you use)

```bash
# If using MySQL
mysqldump -u root -p --opt panel > ./panel.sql

# If using MariaDB
mariadb-dump -u root -p --opt panel > ./panel.sql
```
</Callout>

If you already have Pterodactyl running in Docker, simply update your panel image in your `docker-compose.yml` file:

```diff
-image: ghcr.io/pterodactyl/panel:latest
+image: ghcr.io/blueprintframework/hydrodactyl:main
```

<Callout type="error">
  Moving from Hydrodactyl to Pterodactyl is unsupported and can cause data loss. Hydrodactyl makes changes to the database schema, making it unlikely Pterodactyl will function correctly. If you care about the validity of your database, **make a backup** before swapping back to Pterodactyl.
</Callout>
